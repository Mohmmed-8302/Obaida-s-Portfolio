# Frontend Design: Key Principles

Use these guidelines whenever building or editing any portfolio, website, or visual UI. These principles override generic defaults.

## Core Rules

**1. Distinctive Direction**
Make deliberate, opinionated choices about palette, typography, and layout that are specific to THIS brief. Take one real aesthetic risk you can justify — not a safe, generic choice.

**2. Subject-Grounded Work**
Ground design choices in the actual domain — its materials, tools, and language — rather than generic patterns. A video editor's portfolio should feel like video editing; a dev's portfolio should feel like code.

**3. Typography as Personality**
Type treatment must be memorable and intentional. Pair fonts deliberately, not by grabbing familiar combinations. Typography is the loudest design decision.

**4. Structural Integrity**
Design elements like numbering, spacing, and dividers encode real information about content structure — they are not decoration. Every element must earn its place.

**5. Measured Animation**
Motion serves the subject when orchestrated purposefully. Scattered effects without purpose signal generic AI output. Animate to guide attention, not to impress.

**6. Two-Pass Process**
- **Pass 1**: Plan the system first — color palette, type scale, layout grid, one signature element.
- **Pass 2**: Critique it against the brief before building. Is it genuinely specific to this project, or could it work for any site?

**7. Content Discipline**
Copy is a design material. Write from the user's perspective using clear action language: "Save changes" not "Submit". Every word must justify being there.

**8. Restraint Over Accumulation**
Concentrate boldness in ONE signature element. Keep everything else disciplined. Impact comes from contrast, not accumulation of effects.

## Anti-Patterns to Avoid

Current AI defaults cluster around three looks — reject them unless the brief specifically calls for them:
- Cream/off-white backgrounds with serif body text
- Dark backgrounds with neon/acid accent colors
- Newspaper-style grid layouts

If you're about to reach for one of these, stop and ask: does the brief actually require this, or is it the path of least resistance?

## Checklist Before Finalizing Any UI

- [ ] Does the color palette feel specific to THIS project's domain?
- [ ] Is the typography pairing memorable, not just safe?
- [ ] Is there exactly ONE bold signature element? (not zero, not five)
- [ ] Does every animation serve a purpose?
- [ ] Is the copy written from the user's perspective?
- [ ] Could this design work for a completely different client? (If yes, it's too generic)
